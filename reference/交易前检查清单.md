# 交易前检查清单

这份清单用于下单前快速确认：你是在按计划交易，还是只是因为“看好”而提前入场。

如果只是实盘下单前快速拦截系统外交易，优先使用 [极简交易前检查清单](./极简交易前检查清单.md)。本文件作为完整版，只在需要详细拆解仓位、周期和试仓规则时使用。

## 2. 交易状态

下单前先给这笔交易定性：

- `观察池`：值得关注，但还不能买
- `试仓`：触发未完全完成，只允许小风险参与
- `正式入场`：触发已经完成，可以按计划风险入场
- `放弃`：条件不清楚或风险回报不合适

## 3. 触发条件

必须能写成一句话：

```text
只有当 ______ 发生时，我才允许入场。
```

如果写不出来，说明触发条件还不够清楚。

## 4. 试仓规则

如果触发还没完成，但你仍然想参与：

- 最大风险不超过 `0.25R`
- 不允许因为盘中上涨临时加到完整仓位
- 触发失败就按试仓计划退出
- 只有触发真正完成后，才允许补到完整计划风险

## 6. 防止临场换规则

下单前问自己：

```text
如果这笔交易马上亏损，我会不会改口说“我是做大周期的”？
```

如果会，说明这笔交易的周期和止损规则还没定义清楚。

## 7. 最后一问

```text
这笔交易到底是在验证哪个 setup？
```

只能选一个主 setup：

- 箱体突破
- 杯柄突破
- 均线回踩
- 其他

可以有辅助理由，但不能让辅助理由抢走主规则。

## 提醒

这份清单只用于学习、研究和复盘，不构成投资建议。它的目标不是让每笔交易都盈利，而是减少“没有规则却自以为有规则”的交易。
