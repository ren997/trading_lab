# 0002 R 倍数与仓位风险

## 学习目标

这一课要解决一个很实际的问题：以后你不要只说“这笔赚了 / 亏了多少钱”，而要能说“这笔赚了 / 亏了多少 R”。

`R` 是你在一笔交易中事先愿意承担的计划风险。  
如果一笔交易计划最多亏 1000 元，那么：

- 亏 1000 元 = `-1R`
- 赚 2000 元 = `+2R`
- 亏 500 元 = `-0.5R`

这样做的意义是：不同账户规模、不同市场、不同品种的交易，终于可以放在同一把尺子上比较。

## 为什么现在先学这个

你现在已经不是单纯学习“什么是交易”的阶段。你已经做过币圈、美股和 A 股的主观趋势交易，也提到自己从明显亏损逐渐来到不亏或小赚附近。

这个阶段最容易卡住的地方是：

- 看对方向，但仓位过大，一次回撤打乱节奏
- 胜率不低，但亏损单平均亏太多
- 有盈利单，但太早止盈，导致平均盈利不够覆盖亏损
- 复盘时只记得“这笔赚了”，但不知道这类 setup 长期是否值得继续做

所以接下来要把交易结果统一换算成 `R`。这是你从“凭感觉复盘”进入“可统计复盘”的第一步。

## 必要概念

### 1. 计划风险

计划风险是入场前就定义好的最大可接受亏损。

简单公式：

```text
计划风险金额 = 入场价与止损价之间的差额 × 持仓数量
```

例如：

```text
入场价：100
止损价：95
持仓数量：200 股

计划风险金额 = (100 - 95) × 200 = 1000
```

这笔交易的 `1R = 1000`。

### 2. R 倍数

R 倍数用来表示最终结果相对计划风险的大小。

```text
R 倍数 = 实际盈亏金额 / 计划风险金额
```

例如：

```text
计划风险：1000
最终盈利：2500

R 倍数 = 2500 / 1000 = +2.5R
```

如果最终亏损 800：

```text
R 倍数 = -800 / 1000 = -0.8R
```

### 3. 单笔风险比例

单笔风险比例是你每一笔交易最多愿意亏掉账户的多少。

```text
单笔风险比例 = 计划风险金额 / 账户权益
```

例如：

```text
账户权益：100000
单笔计划风险：1000

单笔风险比例 = 1%
```

这不是建议你必须用某个固定比例，而是提醒你：仓位不是根据“有多看好”随便加，而是要先问“如果错了，我愿意亏多少”。

## 一个最小练习

打开 `01_交易日志与复盘/trade_log_template.csv`，先不要急着填 30 笔。  
你只需要先填最近 5 笔交易，并补齐这些字段：

- `market`
- `symbol`
- `setup`
- `entry_price`
- `stop_price`
- `exit_price`
- `quantity`
- `planned_risk_amount`
- `net_pnl`
- `result_r`
- `followed_plan`
- `main_mistake`

如果有些历史交易数据不完整，可以先估算，但要在 `notes` 里写明“估算”。

## 自测问题

填完 5 笔后，回答下面 5 个问题：

1. 哪一笔交易的亏损超过了 `-1R`？为什么？
2. 哪一笔盈利单其实没有达到原计划的退出条件？
3. 你的最大盈利单来自哪个 setup？
4. 你的最大亏损单是 setup 问题，还是执行问题？
5. 如果下一笔交易只能改一件事，你最该改仓位、止损、入场，还是退出？

## 最容易犯的错

### 错误 1：先买，再想止损

如果你入场后才想止损，计划风险就不是真计划，而是事后补解释。

### 错误 2：用金额判断交易质量

赚 3000 元不一定是好交易，亏 500 元也不一定是坏交易。  
关键要看它是否符合计划、承担了多少风险、结果是多少 R。

### 错误 3：亏损时移动止损

如果你原计划最多亏 `-1R`，结果因为不愿认错亏到 `-2R`，那不是行情问题，而是执行规则失效。

## 下一步建议

本课之后，先不要急着研究更多形态。  
请先完成最近 30 笔交易的日志整理。整理完以后，我们再按 setup 分开看：

- 箱体突破
- 杯柄突破
- 均线回踩
- 其他

然后你就能知道：到底是哪一类 setup 在贡献收益，哪一类在制造回撤。

## 风险提醒

这里的内容只用于学习、研究和模拟验证，不构成个性化投资建议。任何交易都存在不确定性和潜在损失；在没有形成稳定验证流程之前，不应因为短期盈利而扩大仓位、增加杠杆或提高交易频率。
