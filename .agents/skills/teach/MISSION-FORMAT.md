# MISSION.md 格式

`MISSION.md` 位于工作区根目录，用来记录“为什么要学这个主题”。后续每节课教什么、怎么教，都应该能回到这个文件。

## 模板

```md
# Mission: {主题}

## Why
{1 到 3 句话。说明真实世界里的目标，不要只写“我想了解一下”。}

## Success looks like
- {一个可以被观察到的结果}
- {另一个可以被观察到的结果}

## Constraints
- {时间、基础、工具偏好、风险边界等约束}

## Out of scope
- {当前明确不追求的相邻主题}
```

## 规则

- 一个工作区只服务一个主要学习任务。
- 目标要尽量具体，避免空泛表述。
- 如果用户目标变化了，要更新文件，不要让旧目标继续误导后续教学。
- 文件要短，能真正当作指南针使用。
