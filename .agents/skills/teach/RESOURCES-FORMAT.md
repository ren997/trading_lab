# RESOURCES.md 格式

`RESOURCES.md` 用来维护这个主题的高信任度资源。解释概念时，优先从这里取材，而不是依赖含糊的记忆。

## 结构

```md
# {主题} Resources

## Knowledge

- [资源名](https://example.com)
  说明这份资源讲什么，以及适合在什么情况下使用。

## Wisdom (Communities)

- [社区名](https://example.com)
  说明这个社区的价值，以及适合带着什么问题去看。

## Gaps

- 还缺什么资源
```

## 规则

- 优先高信任度来源，例如官方文档、监管机构、权威教材和高质量技术文档。
- 每个条目都要写一行注释，不能只有裸链接。
- 如果某个领域缺资源，要明确写进 `Gaps`。
- 发现资源浅薄、误导或偏题时，应该移除而不是堆积。
